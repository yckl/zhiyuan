# 校园志愿服务管理系统 / Campus Volunteer System

一个面向高校志愿服务场景的 **多角色、双端适配、闭环管理平台**。

它不是单纯的“活动报名后台”，而是把 **志愿者招募、活动审核、现场签到、培训考试、积分激励、社区互动、组织管理、运营审计** 串成一条完整业务链路的系统。

---

## 项目定位

本项目聚焦高校志愿服务数字化管理，覆盖三类核心角色：
- **志愿者端**：报名活动、扫码签到、培训考试、积分商城、心得社区、个人成长数据
- **组织者端**：发起活动、审核报名、现场核销、时长结算、通知下发
- **管理员端**：活动审批、组织者审核、内容治理、商城运营、系统配置、日志审计

相比传统校园管理系统，这个项目更强调两件事：
1. **业务闭环完整**：从报名到签到，从培训到考试，从积分到商城，链路完整
2. **体验不是附属品**：前端不是普通 CRUD 后台，而是刻意做了移动端 App 化、PC 端 SaaS 化的差异化体验

## 项目创新点

## 1. 一套前端代码，做出两种完全不同的使用体验
项目没有停留在简单的响应式布局，而是在 `MainLayout.vue` 中根据角色与设备进行 **DOM 级分流渲染**：

- **志愿者移动端**：更像原生 App，带沉浸式 Header、底部悬浮导航、页面 Push/Pop 转场
- **志愿者 PC 端**：更像现代 SaaS，带顶部导航、宽屏内容区、桌面操作逻辑
- **组织者 / 管理员端**：独立工作台布局，兼顾桌面端侧边栏与移动端抽屉/底部导航

这意味着它不是“同一套页面缩放一下”，而是 **按角色 + 设备重构交互方式**。

---

## 2. 活动现场签到不是普通扫码，而是闭环核销设计
项目里最有辨识度的功能之一，是志愿活动现场签到链路：

- 组织者端生成 **动态签到码 / 二维码**
- 签到码包含 **活动标识、随机 nonce、过期时间**
- 志愿者端通过手机扫码进入签到流程
- 后端在核销时做多重校验：
  - **活动锁**：是不是这个活动
  - **身份锁**：当前用户是不是该活动已报名志愿者
  - **时效锁**：二维码是否过期
  - **状态锁**：是否重复签到 / 是否具备签到资格

这套逻辑比“扫个静态码就成功”更接近真实活动执行场景，也更能体现项目的业务思考深度。

---

## 3. 培训 → 学习 → 考试 → 通过后参与活动，形成准入闭环
项目内置了培训学院模块，不只是展示课程，而是做了完整流程：

- 课程列表与课程详情
- 随机抽题生成试卷
- 在线答题与自动判卷
- 考试记录与学习统计
- 待考提醒
- 课程推荐

也就是说，这个系统不是“先报名再说”，而是可以支持 **高门槛志愿活动的前置培训准入机制**。这点在校园志愿平台里很加分。

---

## 4. 将“志愿服务”做成可持续激励系统，而不是一次性任务平台
项目设计了较完整的激励体系：

- 每日签到
- 连续签到 / 补签
- 积分积累
- 幸运抽奖
- 积分商城兑换
- 我的背包 / 兑换记录

这使平台从“任务管理”延伸到“用户活跃运营”，更像一个真正可持续运行的校园公益平台。

---

## 5. 内容社区 + 内容治理并存
项目除了活动管理，还做了心得分享 / 互动社区能力：

- 发布心得 / 图片
- 点赞 / 收藏 / 评论
- 个人分享记录
- 管理端内容治理

同时，后端加入了：
- **敏感词检测切面**
- **敏感词过滤工具**
- **管理端操作日志 AOP 审计**

也就是说，它不是单纯“能发内容”，而是考虑了 **校园场景下内容安全与审计留痕**。

## 技术亮点

### 前端
- **Vue 3 + TypeScript**
- **Element Plus** 作为 UI 基础组件库
- **Pinia** 状态管理
- **Vue Router** 路由控制
- **Vite** 构建工具
- **GSAP** 用于吉祥物、动效等精细交互
- **html5-qrcode** 实现移动端扫码签到
- **ECharts** 做后台统计与可视化

### 后端
- **Spring Boot 3**
- **Spring Security + JWT** 无状态鉴权
- **MyBatis-Plus** 数据持久化
- **MySQL** 业务数据存储
- **Redis** 缓存与辅助状态管理
- **Knife4j / OpenAPI** 接口文档
- **AOP** 实现操作日志审计
- **敏感词过滤器** 实现社区内容治理

## 快速启动

## 环境要求
- **Node.js** 18+
- **JDK** 17+
- **MySQL** 8+
- **Redis** 6+

---

## 1）启动后端

先创建数据库：

```sql
CREATE DATABASE volunteer_system DEFAULT CHARACTER SET utf8mb4;
```

然后修改：

`backend/src/main/resources/application.yml`

重点配置：
- MySQL 地址、用户名、密码
- Redis 地址
- JWT Secret

启动后端：

```bash
cd backend
mvn spring-boot:run
```

默认地址：
- 后端接口：`http://localhost:8080/api`
- Swagger / Knife4j：`http://localhost:8080/api/swagger-ui.html`

---

## 2）启动前端

```bash
cd frontend
npm install
npm run dev
```

默认地址：
- 前端：`http://localhost:5173`

前端已通过 Vite 代理把 `/api` 转发到后端 `8080`。

如果把它当作课程设计或毕业设计来看，这个项目的完成度是比较高的，优势主要在这几块：

- 过程审计
- 内容治理
- 业务状态流转
